# v0.3 — Windows XP install USB

## Why XP is its own mode

XP install media predates USB-aware Windows installers by a decade. The boot chain is fundamentally different from Vista+:

| Stage                  | Vista / Win 7+ (`--type=windows`)  | Windows XP (`--type=windows-xp`)               |
|------------------------|------------------------------------|------------------------------------------------|
| MBR boot code          | `ms-sys --mbr7`                   | `ms-sys --mbr` (Win 2000/XP/2003 MBR)          |
| FAT32 PBR              | `ms-sys --fat32pe` (loads `bootmgr`) | `ms-sys --fat32nt` (loads `NTLDR`)             |
| Stage-2 loader         | `bootmgr` → loads `BCD` from BCD store | `NTLDR` → reads `boot.ini` → invokes setupldr  |
| Source media expected  | USB or DVD (designed for both)     | CD / floppy (USB requires modifications)       |
| Install file path      | `sources/install.wim`              | `i386/` directory tree                         |

After NTLDR runs, XP's text-mode setup phase tries to load drivers and copy install files. It expects the source media to behave like a CD-ROM. A USB stick *is* a mass-storage device but XP's stock `txtsetup.sif` lists USB drivers only as input devices — they're loaded, but not recognized as boot-bus extenders. The install reaches the "Please wait while Setup loads files..." stage, then the second-stage reboot fails to find the install source.

## The WinSetupFromUSB recipe

This is the canonical solution circa 2007-2009 (Ilko-y, boot-land.net). The modifications are:

### Required: `txtsetup.sif` modification

Move five driver lines from `[InputDevicesSupport.Load]` to `[BootBusExtenders.Load]`:

```
usbehci, usbehci, usbehci.sys
usbohci, usbohci, usbohci.sys
usbuhci, usbuhci, usbuhci.sys
usbhub,  usbhub,  usbhub.sys
usbstor, usbstor, usbstor.sys
```

Also add corresponding entries in `[BootBusExtenders]`:

```
usbehci  = "USB 2.0 Enhanced Host Controller"
usbohci  = "USB Open Host Controller"
usbuhci  = "USB Universal Host Controller"
usbhub   = "USB Standard Hub"
usbstor  = "USB Mass Storage"
```

Without this, text-mode setup loses the USB source after the reboot.

### Recommended: `WaitBT.sys` and `Wait4UFD.sys` drivers

Third-party waiters that delay the kernel until the BIOS finishes initializing USB on the target machine. Without them, some hardware throws `0x7B INACCESSIBLE_BOOT_DEVICE` on the post-text-mode reboot.

Provenance: originally written by ilko-y for boot-land.net (2009). Free for redistribution but not GPL or MIT-licensed. usbwin's approach for v0.3: **user-provided via `--xp-waiters /path/to/drivers/`** so we don't bundle and don't have to argue about license status.

### Optional: `winnt.sif` (answer file)

If present at the partition root, XP setup reads it to skip prompts (EULA, partitioning, product key, computer name, etc.). usbwin generates a minimal `winnt.sif` if the user supplies `--xp-unattended`.

## NT 3.x / NT 4.0 — out of scope

Pre-2000 NT family doesn't support USB at all in the installer phase. NT 3.x has zero USB support; NT 4.0 had third-party USB drivers but never first-class. Installing NT 3.x / 4.0 from USB is essentially impossible without VM-then-image-the-disk shenanigans. Out of scope for usbwin.
